# System Specification: ErgoFlow AI
**Target Hackathon Track:** Google Cloud Rapid Agent Hackathon (MongoDB Track)
**Implementation Target:** Autonomous Context-Aware Occupational Health Agent

---

## 1. Project Overview & Intent

### 1.1 Problem Statement
Desktop-bound software professionals frequently suffer from repetitive strain injuries, posture degradation, and physical fatigue due to uninterrupted focus sessions. Existing solutions rely on static timer alerts that interrupt deep-work states arbitrarily, leading to high user dismissal rates and a lack of hyper-personalized physical recovery.

### 1.2 The Solution
`ErgoFlow AI` is an autonomous, stateful health orchestration agent. By utilizing **Google Cloud Agent Builder** and **Gemini 3**, it continuously analyzes an engineer's real-time workflow calendar, passive biometric telemetry (Apple HealthKit / Google Fit), and periodic 1-click qualitative micro-prompts. Instead of acting as a passive reminder system, the agent reasons over historical data states to actively schedule micro-recovery blocks directly into the user’s calendar, serving customized mobility routines optimized for their specific physical fatigue signatures.

---

## 2. System Architecture Specification

An implementation agent can realize this application using a decoupled, event-driven architecture structured as follows:

* **Frontend (UI Layer):** A single-page application built using **ReactJS** or **Vue.js**. This application acts as the user dashboard (visualizing historical mobility indices) and renders low-friction modal dialogs for qualitative state check-ins.
* **Backend (Orchestration & Integration Layer):** A **Java / Spring Boot** microservice framework. This handles OAuth2 authentication for Google Calendar APIs, ingests time-series biometric streams from Apple HealthKit / Health Connect webhooks, and exposes REST endpoints for the frontend.
* **Database & Memory (State Layer):** A **MongoDB Atlas** cluster exposed to the AI model via the official **MongoDB Model Context Protocol (MCP) Server**. This acts as the long-term memory system for correlations and historical health logs.
* **Cognitive Layer:** **Google Cloud Agent Builder** orchestrating a **Gemini 3** model, configured with tool call capabilities to execute calendar mutations and query MongoDB collections via natural language abstraction.

---

4. Google Cloud Agent Builder & Gemini 3 Prompt Specification
Copy-paste this configuration payload directly into the Google Cloud Agent Builder system instructions.

# SYSTEM PERSONA
You are ErgoFlow AI, an autonomous, highly specialized occupational health and workplace ergonomics optimization agent designed specifically for software engineers. Your goal is to analyze quantitative and qualitative metrics to protect the user's physical well-being without disrupting critical work flows.

# OPERATIONAL PROTOCOLS
1. CONTEXT ANALYSIS: Whenever the system triggers an evaluation loop or the user asks for an assessment, execute tool queries via the MongoDB MCP to fetch the latest documents from `biometric_telemetry` and `subjective_feedback`.
2. CORRELATION REASONING: 
   - IF consecutive_sitting_mins >= 90 AND lower_back_stiffness >= 3, flag a high posture-degradation event.
   - IF stand_hours_today < 3 AND timestamp indicates it is past 14:00, flag an acute physical inactivity event.
3. AUTONOMOUS INTERVENTION: Upon detecting a degradation or inactivity event, check the user's upcoming calendar availability using the Google Calendar Tool integration.
4. CALENDAR MUTATION: Find the first available open 10-to-15 minute window immediately following a deep work block or high-stress meeting block. Inject a calendar event entitled "ErgoFlow: Optimized Recovery Protocol".
5. PROTOCOL REGENERATION: Generate a highly tailored JSON array of movements inside the `orchestrated_routines` collection matching the specific physical strain points discovered during step 2. Do not emit generic guidelines.

5. Core API Endpoint Matrix (Spring Boot Controller Blueprint)
The Spring Boot backend must implement the following REST endpoints to interface between the React/Vue frontend, Google Cloud webhooks, and MongoDB Atlas.

POST,/api/feedback/micro-prompt,SubjectiveFeedbackDTO JSON,Receives 1-click physical degradation scores directly from frontend modals.

To execute this project step-by-step, follow this sequence:

Phase 1: Environment Setup & Data Backbone
Initialize a Java Spring Boot backend project incorporating spring-boot-starter-data-mongodb and security packages.

Spin up a shared M0 cluster on MongoDB Atlas and populate seed data using the schemas in Section 3.

Build a simple ReactJS or Vue.js boilerplate dashboard containing layout cards for tracking state.

Phase 2: Integration & Telemetry Hookup
Connect the Spring Boot backend to Google Calendar using the official Google API Client Library to enable calendar scanning and scheduling operations.

Implement dummy endpoints mimicking Apple HealthKit JSON payloads so telemetry can be updated locally via standard POST requests.

Build the micro-prompt UI component in the frontend—a slide-out modal presenting five-point sliders for muscle pain and visual fatigue.

Phase 3: MCP & AI Agent Orchestration
Deploy the official MongoDB MCP Server and map it to your active Atlas connection strings.

Inside Google Cloud Agent Builder, instantiate a new Agent backed by Gemini 3. Register the MongoDB MCP instance as an active capability tool.

Input the Prompt Specification from Section 4 into the system instructions panel of your Agent Builder dashboard.

Phase 4: Verification Loop & Demo Assembly
Perform an end-to-end simulation: Push a data state representing 120 minutes of continuous coding and high fatigue into MongoDB. Check if Gemini autonomously provisions a 10-minute slot on your connected Google Calendar.
